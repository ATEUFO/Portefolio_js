Manifeste technique

Du besoin métier au système qui tourne.

Nous combinons orchestration (n8n), agents IA (OpenClaw), protocoles ouverts (MCP) et design d’API pour que l’automatisation reste auditée, sécurisée et maintenable.

olenceia / orchestration.ts
import { pipeline } from '@olenceia/core';
import { n8n, openclaw, mcp } from '@olenceia/connectors';
 
export async function bootstrap() {
const flow = await n8n.compose({
triggers: ['crm', 'ticketing', 'webhook'],
ai: openclaw.agent({ memory: 'hybrid' }),
tools: mcp.bridge(12)
});
return flow.deploy({ region: 'eu' });
}

Services

Ce que nous livrons concrètement.

Des blocs modulaires — vous pouvez démarrer par un workflow critique, puis étendre vers l’agentique et le multi-service.

Workflows n8n

Conception de scénarios fair-code : intégration CRM, facturation, support, e-mail et bases de données. Nœuds IA pour enrichir ou classer sans quitter l’orchestrateur. Idéal pour un time-to-value rapide avec traçabilité des exécutions.

Agents OpenClaw

Mise en place d’agents IA orientés tâches : mémoire structurée, outils (dont MCP), routage multi-modèles et canaux (messagerie, webhooks). Approche local-first possible pour les données sensibles, avec politiques de secrets et sandboxing — aligné sur les pratiques production 2026 (supervision, garde-fous réseau).

MCP & outils

Ponts MCP pour brancher vos bases, tickets et APIs sur des modèles conversationnels — le bon niveau d’abstraction entre « prompt » et « système d’information ».

Chatbots métiers

Assistants connectés à votre réalité opérationnelle : stocks, commandes, SLA, procédures internes. Réponses contextualisées, escalade humaine, journaux d’audit.

Apps & microservices

Développement d’applications d’entreprise découpées en services, contrats d’API explicites, déploiements conteneurisés et scalabilité horizontale — prêt pour les équipes DevOps.

MCP : standardiser l’accès des modèles à vos outils métiers.
IA comme co-pilote : l’humain valide, le système exécute.

Architecture

Microservices, messages, résilience.

Nous dessinons des frontières claires entre domaines : chaque service expose une API stable, les événements circulent sur des bus maîtrisés, et l’observabilité (logs, métriques, traces) n’est pas un ajout de fin de projet.

Principes

  • Découplage : moins de déploiements monolithiques bloquants.
  • Contrats : schémas OpenAPI, versioning, compatibilité ascendante.
  • Sécurité : auth fédérée, moindre privilège, secrets hors du code.

Livrables types

Cartographie des flux, runbooks, pipelines CI/CD, environnements de staging — pour que l’équipe IT reprenne le flambeau sans reverse-engineering.

Angle mort à éviter
« Belle démo » sans monitoring ni ownership des données — nous priorisons l’exploitable.
